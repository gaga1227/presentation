Develop for Touch Friendliness

About

Human behaviour changes rapidly and foundamentally during last decade thanks to new technology and an explosion of devices.

  • Touch (iPhone, Android devices)
  • Gesture (Wii, Kinetic)
  • Voice (Siri)

Touch for Web

One of the key interaction type afforded by those web capable devices are touch interactivity. So big of a impact that it's part of our daily human behaviour to expect everything can be interacted with touch.

As this human behavior becomes more dominant in web, we should prepare ourselves, or even optimize towards touch inputs, so our audiences has a better experience.

Current Situation on the Web

Altough touch input method is becoming more popular, things are not so promising on the web. Just try using you device to do all the web surfing for a week and you'll know how bad it is.

Problems

  1. Legacy - most websites done until recently almots never consider touch inputs
  2. Backward compatibility - due to the issue above, device engineers decided to provide legacy support, which to some degree hinders web developers and businesses to invest in optimising for new input methods.

What we can do NOW?

  1. Seprarate input methods and cater individually (CSS, JS)
  2. Optimize for devices, generally
  3. Fix or workaround known bugs/issues

CSS

  • Modernizr
  • Hover 
    .no-touch :hover,
.no-touch :focus {
    /* your hover/focus styles */
}
.touch :active,
.touch .active {
    /* your tap down styles */
}

JS Helpers

  • Zepto.js
  • Hammer.js
  • FastClick
  • FastActive.js
  • and a lot more...

The Road Ahead

  • Design and develop for dominant input method (touch, multi-touch?)
  • But mouse input compatible
  • Ever increasingly complex web requires more resource spent on UX because you'll never cover it all

Summary

  1. Need to support multiple input methods, optimize project by project
  2. Change the way you handle styles (CSS)
  3. Change the way you handle interactions (JS)
  4. Get helps from community
