Touch Friendliness for Web

WHY?

  1. Most web browsing capable devices now mainly support touch inputs
  2. Devices are more accessible and
    touch feels more natural to us
  3. Touch inputs will soon dominate the web
    (in non-office environments)

So, how touch friendly is web now?

Touch? No~~~

Problems

  1. Legacy - website updates can't keep up with new technologies
  2. Backward compatibility - new tech engineers decided to provide legacy support
  3. Vicious Cycle - web businesses and web developers feel no need investing in new tech support

Break out of the cycle

if:

We value our customers' experience and
We provide better services with new techs

by:

Supporting both traditional and touch inputs while catering for them individually

CSS

Hover with Modernizr and FastActive 

.no-touch :hover,
.no-touch :focus {
  /* your hover/focus styles */
}
.touch :active,
.touch .active {
  /* your tap down styles */
}

JS

Input handler with Modernizr and Zepto 

var _input;
if (Modernizr.touch) {
  _input = 'tap';
} else {
  _input = 'click';
}
$btn.on(_input, function(e){
  // your button actions
});
